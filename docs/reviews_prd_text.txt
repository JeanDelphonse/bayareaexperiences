1. Feature Overview
1.1 Purpose
This PRD defines the requirements for a customer feedback and review system for bayareaexperiences.com. After completing a tour, customers receive an automated follow-up email with a direct feedback link. They provide a 1–5 star rating and answer one focused question: "What was the single best moment of your experience?" Reviews are published on the corresponding experience detail page at the bottom, building social proof that drives future bookings.
1.2 Design Decisions
Decision
Rationale
One rating (stars) + one question only
Higher completion rate than multi-question surveys. The 'best moment' question yields emotional, specific, quotable content — far more persuasive than generic ratings.
Verified bookers only
Prevents fake reviews. Every review is backed by a real booking record, which is displayed to admin and optionally to customers as a 'Verified Experience' badge.
Guest bookers supported via token link
Guests without accounts represent a significant share of bookings. Excluding them from feedback would bias the review pool toward account holders.
Auto-publish with 24-hour hold on 1–2 star reviews
Instant publishing maximizes review volume and customer trust. The 24-hour hold on very low scores gives admin time to review and optionally reach out to the customer before the negative review goes live.
Provider response schema in DB, UI deferred
The marketplace model (BAE-PRD-MARKETPLACE) means providers will eventually want to respond to reviews publicly. Building the schema now avoids a future migration.
3-hour post-tour email trigger
Customers are home, relaxed, and still emotionally warm. Completion rates peak at 2–4 hours post-experience. Earlier feels rushed; later the moment has faded.
Display: newest first, max 10 per page
Freshest reviews are most relevant. Pagination prevents detail page from becoming infinitely long.
1.3 Scope
New page: /feedback/<token> — one-screen feedback form for verified bookers
Updated page: /experience/<slug> — review display section appended at bottom
New admin section: /admin/reviews — moderation, flagging, and analytics
New email template: post_tour_feedback.html — sent 3 hours after timeslot end time
New background job: send_feedback_requests() — triggered by nightly scheduler or APScheduler
5 new database tables: experience_reviews, review_tokens, review_votes, review_flags, review_responses

2.6 Review Status Flow
Status
Description / Trigger
pending
Initial state on submission. Not yet visible publicly. Transitions immediately to published or held.
published
Visible on the experience detail page. For 3–5 star reviews: set immediately on submission. For 1–2 star reviews: set after held_until timestamp passes (24 hours).
held
Automatically set for 1–2 star reviews. System schedules transition to published after 24 hours. Admin may intervene during this window: publish early, flag, or remove.
flagged
Admin or automated flag threshold (3+ customer flags) triggers this. Hidden from public. Requires admin review before any action.
removed
Admin permanently removes the review. Not shown publicly. Soft delete — record remains in DB with status=removed.
3. Feedback Collection — Post-Tour Email
3.1 Trigger Logic
The feedback email is sent automatically 3 hours after the timeslot's scheduled end time. A background job checks for eligible bookings every 30 minutes and dispatches emails as they become due.
# app/reviews/scheduler.py
from datetime import datetime, timezone, timedelta
from app.models import Booking, Timeslot, ReviewToken
from app.extensions import db, mail
from app.utils import generate_pk
from app.reviews.email import send_feedback_request
import secrets

def process_pending_feedback_requests():
    """
    Called every 30 minutes (cron or APScheduler).
    Finds completed bookings whose timeslot ended 3+ hours ago
    and where no feedback token has been sent yet.
    """
    now = datetime.now(timezone.utc)
    cutoff = now - timedelta(hours=3)   # 3 hours ago
    max_age = now - timedelta(days=7)   # Don't email tours older than 7 days

    # Find eligible bookings: confirmed + timeslot ended 3–7 days ago + no token sent
    eligible = (
        Booking.query
        .join(Timeslot, Booking.timeslot_id == Timeslot.timeslot_id)
        .filter(
            Booking.booking_status == 'confirmed',
            Timeslot.slot_end_datetime <= cutoff,     # Tour has ended
            Timeslot.slot_end_datetime >= max_age,    # Not too old
            ~Booking.review_tokens.any(),             # No token sent yet
        )
        .all()
    )

    for booking in eligible:
        # Generate unique 64-char token
        token_str = secrets.token_hex(32)  # 64 hex chars
        expires   = now + timedelta(days=30)

        token = ReviewToken(
            token_id    = generate_pk(),
            token       = token_str,
            booking_id  = booking.booking_id,
            experience_id = booking.experience_id,
            user_id     = booking.user_id,
            email_sent_to = booking.guest_email or booking.user.email,
            expires_at  = expires,
        )
        db.session.add(token)
        db.session.flush()  # Get token_id before commit

        # Send email
        success = send_feedback_request(booking, token)
        if success:
            token.email_sent_at = now

    db.session.commit()

# app/blueprints/reviews/views.py
from flask import render_template, redirect, url_for, request, abort
from app.models import ReviewToken, ExperienceReview
from app.extensions import db
from datetime import datetime, timezone

@bp.route('/feedback/<token>', methods=['GET','POST'])
def feedback_form(token):
    # Look up the token
    rt = ReviewToken.query.filter_by(token=token).first()

    # Reject invalid, expired, or already-used tokens
    if not rt:
        abort(404)  # Token not found
    if rt.is_used:
        return render_template('reviews/already_submitted.html', experience=rt.experience)
    if rt.expires_at < datetime.now(timezone.utc):
        return render_template('reviews/token_expired.html',
                               experience=rt.experience,
                               phone='(408) 831-2101')

    booking     = rt.booking
    experience  = rt.experience

    if request.method == 'POST':
        return process_feedback_submission(rt, booking, experience)

    # Pre-fill rating from email star link if provided
    prefill_rating = request.args.get('rating', type=int)
    if prefill_rating and not (1 <= prefill_rating <= 5):
        prefill_rating = None

    return render_template('reviews/feedback_form.html',
                           booking=booking,
                           experience=experience,
                           prefill_rating=prefill_rating)

4.5 Form Submission Processing
# app/blueprints/reviews/views.py  (continued)
from app.models import ExperienceReview, Experience
from datetime import datetime, timezone, timedelta

def process_feedback_submission(rt, booking, experience):
    star_rating  = request.form.get('star_rating', type=int)
    best_moment  = request.form.get('best_moment', '').strip()

    # Server-side validation
    errors = []
    if not star_rating or not (1 <= star_rating <= 5):
        errors.append('Please select a star rating.')
    if len(best_moment) < 10:
        errors.append('Please write at least 10 characters about your best moment.')
    if len(best_moment) > 1000:
        errors.append('Your response must be 1000 characters or fewer.')
    if errors:
        return render_template('reviews/feedback_form.html',
                               booking=booking, experience=experience,
                               errors=errors)

    # Determine publish status — hold 1–2 star reviews for 24 hours
    now = datetime.now(timezone.utc)
    if star_rating <= 2:
        status     = 'held'
        held_until = now + timedelta(hours=24)
        published_at = None
    else:
        status     = 'published'
        held_until = None
        published_at = now

    # Build display name from booking
    first  = booking.guest_first_name or (booking.user.first_name if booking.user else 'Guest')
    last_i = (booking.guest_last_name or (booking.user.last_name if booking.user else ''))[0:1].upper()
    display_name = f'{first} {last_i}.' if last_i else first

    review = ExperienceReview(
        review_id              = generate_pk(),
        booking_id             = booking.booking_id,
        experience_id          = experience.experience_id,
        user_id                = booking.user_id,
        provider_id            = experience.provider_id,
        star_rating            = star_rating,
        best_moment            = best_moment,
        reviewer_first_name    = first,
        reviewer_last_name_initial = last_i,
        reviewer_display_name  = display_name,
        status                 = status,
        published_at           = published_at,
        held_until             = held_until,
        feedback_token_id      = rt.token_id,
        ip_address             = request.remote_addr,
    )
    db.session.add(review)

    # Mark token as used
    rt.is_used  = True
    rt.used_at  = now

    # Update experience aggregate stats if publishing immediately
    if status == 'published':
        update_experience_rating(experience.experience_id)

    db.session.commit()

    # Notify admin of 1–2 star held reviews
    if status == 'held':
        notify_admin_low_score(review, booking, experience)

    return redirect(url_for('reviews.thankyou', token=rt.token))


def update_experience_rating(experience_id):
    """Recompute avg_star_rating and review_count on the experiences table."""
    from sqlalchemy import func
    result = db.session.query(
        func.avg(ExperienceReview.star_rating).label('avg'),
        func.count(ExperienceReview.review_id).label('cnt'),
    ).filter(
        ExperienceReview.experience_id == experience_id,
        ExperienceReview.status == 'published',
    ).one()

    db.session.query(Experience).filter_by(
        experience_id=experience_id).update({
            'avg_star_rating': round(float(result.avg), 2) if result.avg else None,
            'review_count':    result.cnt,
    })

from app.models import ExperienceReview
from sqlalchemy import func

@bp.route('/experience/<slug>')
def experience_detail(slug):
    experience = Experience.query.filter_by(
        slug=slug, is_active=True).first_or_404()

    # Fetch first page of published reviews — newest first
    reviews = ExperienceReview.query.filter_by(
        experience_id=experience.experience_id,
        status='published',
    ).order_by(
        ExperienceReview.is_featured.desc(),  # Featured reviews first
        ExperienceReview.published_at.desc(),
    ).limit(10).all()

    has_more_reviews = ExperienceReview.query.filter_by(
        experience_id=experience.experience_id,
        status='published',
    ).count() > 10

    # Star distribution for bar chart
    star_counts = {i: 0 for i in range(1,6)}
    rows = db.session.query(
        ExperienceReview.star_rating,
        func.count(ExperienceReview.review_id),
    ).filter_by(
        experience_id=experience.experience_id,
        status='published',
    ).group_by(ExperienceReview.star_rating).all()
    for rating, count in rows:
        star_counts[rating] = count

    return render_template('main/experience_detail.html',
                           experience=experience,
                           reviews=reviews,
                           star_counts=star_counts,
                           has_more_reviews=has_more_reviews)

6.3 Low-Score Admin Notification
When a 1–2 star review is submitted and held, BAE admin receives an immediate email notification so they can optionally reach out to the customer before the review goes live.
# app/reviews/notifications.py
def notify_admin_low_score(review, booking, experience):
    """
    Email admin when a 1–2 star review is received.
    Gives admin 24 hours to review before it auto-publishes.
    """
    from app.extensions import mail
    from flask_mail import Message
    import os

    admin_email = os.environ.get('ADMIN_EMAIL')
    if not admin_email:
        return

    msg = Message(
        subject=f'[BAE Review Alert] {review.star_rating}-star review received — {experience.name}',
        recipients=[admin_email],
        body=(
            f'A {review.star_rating}-star review was submitted for {experience.name}.\n\n'
            f'Reviewer: {review.reviewer_display_name}\n'
            f'Tour date: {booking.timeslot.slot_date}\n'
            f'Best moment response:\n"{review.best_moment}"\n\n'
            f'This review is HELD for 24 hours (auto-publishes at {review.held_until} UTC).\n'
            f'Review it at: {admin_review_url}\n\n'
            f'You may publish early, flag for review, or remove it from the admin panel.'
        )
    )
    try:
        mail.send(msg)
    except Exception as e:
        import logging
        logging.getLogger('reviews').error(f'Admin low-score notification failed: {e}')
6.4 Auto-Publish Held Reviews Job
# Part of the nightly/30-min scheduler
def auto_publish_held_reviews():
    """Publish any held reviews whose 24-hour window has passed."""
    now     = datetime.now(timezone.utc)
    held    = ExperienceReview.query.filter(
        ExperienceReview.status == 'held',
        ExperienceReview.held_until <= now,
    ).all()

    for review in held:
        review.status       = 'published'
        review.published_at = now
        update_experience_rating(review.experience_id)

    db.session.commit()
    return len(held)  # Number published in this run
